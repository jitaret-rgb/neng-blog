## 📍 **Step 1: ระบุปัญหา**

### **คำถาม 4 ข้อที่ต้องตอบ:**

```
❓ 1. ปัญหาคืออะไร?
❓ 2. ใครได้รับผลกระทบ?
❓ 3. ปัจจุบันแก้ยังไง?
❓ 4. AI ช่วยอะไรได้?
```

---

### **ตัวอย่างที่ 1: ชาวบ้านถามเรื่องสวัสดิการ**

**1. ปัญหาคืออะไร?**
```
ชาวบ้านโทรถามเรื่องเบี้ยยังชีพผู้สูงอายุวันละ 50+ สาย
```

**2. ใครได้รับผลกระทบ?**
```
- ผู้สูงอายุ (รอนาน)
- เจ้าหน้าที่ (รับไม่ไหว)
- นายก อบต. (ชาวบ้านบ่น)
```

**3. ปัจจุบันแก้ยังไง?**
```
- รับโทรศัพท์
- ตอบ LINE
- ลงพื้นที่อธิบาย
```

**4. AI ช่วยอะไรได้?**
```
Chatbot ตอบคำถามพื้นฐานอัตโนมัติ 24/7
```

---

### **ตัวอย่างที่ 2: ทำรายงานประชุมไม่ทัน**

**1. ปัญหาคืออะไร?**
```
ประชุมบ่อย ทำรายงานไม่ทัน ต้องใช้เวลา 3 ชม./ครั้ง
```

**2. ใครได้รับผลกระทบ?**
```
- เจ้าหน้าที่ (เหนื่อย)
- หัวหน้า (รายงานช้า)
- ที่ประชุม (รอรายงาน)
```

**3. ปัจจุบันแก้ยังไง?**
```
- จดมือ
- พิมพ์เอง
- ทำงานล่วงเวลา
```

**4. AI ช่วยอะไรได้?**
```
สรุปการประชุมอัตโนมัติ + แยก Action Items
```

## 🎯 **Step 2: เลือก Use Case**

### **Use Case คืออะไร?**

```
Use Case = "งานเฉพาะที่ AI จะทำให้"
```

**ไม่ใช่:** "ใช้ AI ทำทุกอย่าง"  
**แต่คือ:** "ใช้ AI ทำ **หนึ่งเรื่อง**"

---

### **ตาราง Use Cases สำหรับงานชุมชน:**

| หมวดหมู่ | Use Case | ความยาก | ผลกระทบ | แนะนำ |
|----------|----------|--------|---------|-------|
| **Conversational** | Chatbot ตอบคำถาม | ⭐⭐ | สูง | ✅ เริ่มที่นี่ |
| **Writing** | สร้างโพสต์ Facebook | ⭐ | ปานกลาง | ✅ ง่าย |
| **Summarization** | สรุปการประชุม | ⭐⭐ | ปานกลาง | ✅ คุ้มค่า |
| **Data Extraction** | ดึงข้อมูลแบบสำรวจ | ⭐⭐⭐ | สูง | ⭐ ปานกลาง |
| **Workflow** | จัดกิจกรรมชุมชน | ⭐⭐⭐ | สูง | ⭐ ยาก |

---

### **แนะนำสำหรับมือใหม่:**

**เริ่มจาก:**
1. ✅ **Chatbot** — ง่าย เห็นผลเร็ว
2. ✅ **Summarization** — คุ้มค่า ประหยัดเวลา

**ยังไม่แนะนำ:**
- ❌ Data Extraction — ยาก ต้องมีข้อมูลเยอะ
- ❌ Workflow — ซับซ้อน ต้อง integrate หลายระบบ

## 📏 **Step 3: กำหนดขอบเขต**

### **ทำไมต้องกำหนดขอบเขต?**

```
❌ ไม่กำหนด → AI พยายามทำทุกอย่าง → ผลลัพธ์แย่
✅ กำหนดชัด → AI ทำ **หนึ่งเรื่อง** → ผลลัพธ์ดี
```

---

### **ตัวอย่าง: Chatbot สวัสดิการ**

**✅ In Scope (ทำอะไร):**
```
- ตอบคำถามเบี้ยยังชีพผู้สูงอายุ
- ตอบคำถามบัตรสวัสดิการ
- ตอบคำถามโครงการเงินกู้ กทบ.
- เวลาทำการ: 24/7
- ภาษา: ไทย
```

**❌ Out of Scope (ไม่ทำอะไร):**
```
- เรื่องการเมือง
- เรื่องส่วนตัว
- ข้อมูลความลับ
- กรณีฉุกเฉิน (ให้โทร 191)
- เรื่องนอกเหนือจากสวัสดิการ
```

---

### **Send-off Strategy (ส่งต่อมนุษย์):**

```
เมื่อไหร่ให้ส่งต่อมนุษย์?

1. คำถามซับซ้อน
   → "กรุณารอเจ้าหน้าที่ติดต่อกลับ"

2. ข้อมูลไม่ชัดเจน
   → "ขอตรวจสอบข้อมูลเพิ่มเติม"

3. เรื่องอารมณ์/ความเครียด
   → "ขอแนะนำปรึกษาเจ้าหน้าที่โดยตรง"

4. กรณีฉุกเฉิน
   → "กรุณาโทร 191 ทันที"
```

### **เปรียบเทียบโมเดลยอดนิยม:**

| โมเดล | ราคา | ภาษาไทย | Context | เหมาะกับ |
|-------|------|---------|---------|---------|
| **Qwen 2.5** | ฟรี 1M tokens/เดือน | ⭐⭐⭐⭐ | 32K | ✅ Chatbot ทั่วไป |
| **GPT-4o-mini** | $0.15/1M tokens | ⭐⭐⭐⭐ | 128K | ✅ งบจำกัด |
| **Claude Sonnet** | $3/1M tokens | ⭐⭐⭐⭐⭐ | 200K | ✅ Content |
| **Gemini 2.0** | ฟรี 60 requests/min | ⭐⭐⭐⭐ | 1M | ✅ ทดลอง |
| **Llama 3** | ฟรี (host เอง) | ⭐⭐⭐ | 8K | 🔒 ข้อมูลลับ |

---

### **แนะนำสำหรับแต่ละ Use Case:**

| Use Case | โมเดลแนะนำ | เหตุผล |
|----------|-----------|--------|
| **Chatbot** | Qwen 2.5 | ฟรี ภาษาไทยดี |
| **Content** | Claude Sonnet | ภาษาธรรมชาติ |
| **Summarization** | Gemini 2.0 | ฟรี Context ยาว |
| **Data Analysis** | GPT-4o-mini | ถูก แม่นยำ |
| **ข้อมูลลับ** | Llama 3 | Host เอง ปลอดภัย |

---

### **💰 เปรียบเทียบราคา:**

```
**สมมติ:** ตอบคำถามวันละ 100 คำถาม
      คำถามละ 500 tokens
      รวม: 50,000 tokens/วัน = 1.5M tokens/เดือน

Qwen 2.5:     ฟรี (1M tokens แรก)
              ส่วนเกิน: ~$1-2/เดือน

GPT-4o-mini:  1.5M x $0.00015 = $0.225/เดือน
              (~8 บาท/เดือน!)

Claude Sonnet: 1.5M x $0.003 = $4.5/เดือน
               (~160 บาท/เดือน)

สรุป: ถูกมาก! เริ่มต้นที่ 0-200 บาท/เดือน
```

## 💡 **ตัวอย่าง: Chatbot สวัสดิการชุมชน**

> **⚠️ หมายเหตุ:** นี่เป็น **ตัวอย่างสมมติ** เพื่อประกอบการเรียนรู้  
> ชื่อสถานที่และข้อมูลเป็นเพียง **ตัวอย่าง** เพื่อให้เห็นภาพชัดเจน  
> สามารถนำไปปรับใช้กับหน่วยงานจริงได้

### **ข้อมูลโครงการ (สมมติ):**

```
📍 สถานที่: อบต.ตัวอย่าง (สมมติ) จ.ตัวอย่าง (สมมติ)
📅 เวลา: มกราคม - มีนาคม 2569
👥 ทีม: เจ้าหน้าที่ 3 คน
```

---

### **ปัญหา:**

```
❌ ชาวบ้านโทรถามเรื่องเบี้ยยังชีพวันละ 50+ สาย
❌ เจ้าหน้าที่ 3 คน รับไม่ไหว
❌ ชาวบ้านรอนาน บางทีไม่ได้คำตอบ
❌ เจ้าหน้าที่ไม่มีเวลาทำโครงการใหม่
```

---

### **วิธีแก้:**

**1. รวบรวม FAQ**
```
- รวบรวมคำถามพบบ่อย 50 คำถาม
- เขียนคำตอบมาตรฐาน
- แบ่งหมวดหมู่ (เบี้ยยังชีพ, บัตรสวัสดิการ, กทบ.)
```

**2. สร้าง Chatbot**
```
- ใช้ Qwen 2.5 (ฟรี)
- ใช้ RAG (Retrieval-Augmented Generation)
- Deploy บน LINE (ชาวบ้านคุ้นเคย)
```

**3. ทดสอบ**
```
- ทดสอบกับ อสม. 10 คน
- แก้ไขคำตอบผิด
- เพิ่มคำถามใหม่
```

**4. เปิดใช้งาน**
```
- เปิดใช้งานจริง
- มีปุ่ม "ติดต่อเจ้าหน้าที่"
- ติดตามการใช้งาน
```

---

### **ผลลัพธ์ (หลัง 3 เดือน) - ตัวอย่างสมมติ:**

> **⚠️ หมายเหตุ:** ตัวเลขนี้เป็น **ตัวอย่างสมมติ** เพื่อแสดงให้เห็นถึงความเป็นไปได้  
> ผลลัพธ์จริงอาจแตกต่างกันไปตามบริบทของแต่ละพื้นที่

| ตัวชี้วัด | ก่อน | หลัง | เปลี่ยนแปลง |
|----------|------|------|-------------|
| **สายโทรเข้า** | 50 สาย/วัน | 15 สาย/วัน | 📉 -70% |
| **เวลารอคำตอบ** | 30 นาที | 0 นาที | ⚡ ทันที |
| **เวลาเจ้าหน้าที่** | 4 ชม./วัน | 1 ชม./วัน | ⏱️ -75% |
| **โครงการใหม่** | 0 โครงการ | 3 โครงการ | 📈 +3 |
| **ความพึงพอใจ** | 60% | 90% | 😊 +30% |

```

### **People Checklist:**

```
□ หัวหน้าสนับสนุน
